Configuration

Les donnes sources, cest plus de 1500 indicateurs (mtriques) par communes.
Par dfaut, un certain nombre de donnes sont collectes.
Par configuration, on peut ensuite demander la collecte de donnes supplmentaires.
On peut aussi calculer de nouvelles mtriques bases sur les autres (par exemple des taux)

Ces mtriques sont ensuite disponibles pour gnrer le rapport.

Pour collecter de nouvelles mtriques, ajouter des lignes dans le fichier de configuration, TAB Collect

Key : Lidentifiant de la mtrique (donne)

Ce champ doit tre unique, sans espaces ni caractres spciaux. Exemple :

SRU_CARENCE_2020

Description : La Description de la mtrique (indicateur de la donne)

Texte libre, par exemple :

Taux de Carence en 2020

Source: La Source de la mtrique

Ce champ est indicatif, mais il permet de classer les metriques dans linterface.

  • DATA : Donnes de Base
  • CODE : Donnes Codes Postaux
  • INTERCO : Donnes Intercommunalits
  • INSEE : Donnes INSEE
  • CALC : Donnes calcules
  • SRU : Donnes SRU DREAL
  • ART : Donnes Artificialisation CEREMA
  • PROJ : Donnes Projection INSEE
  • EVOL : Donnes Evolution INSEE
  • SIT ; Donnes SITADEL

Type : Le type de la mtrique

Les types supports sont :

  • INT : Entier sans dcimale
  • STR : Chaine de caractres
  • FLOAT : Nombre dcimal
  • TAUX : Taux. 0,1 reprsente 10 %
  • PERCENT : Pourcentage. 0,1 reprsente 0,1 %

Data : Le calcul de la mtrique (une expression Python)

Total : Le calcul du total de la mtrique pour plusieurs communes

Les possibilits sont :

  • SUM : la somme des valeurs de toutes les communes
  • AVG : la moyenne des valeurs de toutes les communes
  • COUNT : le nombre de communes
  • EQUAL : la mme valeur de la premire commune
  • IGNORE : Ignore le total
  • N/A : Non-applicable
  • CUSTOM : une expression python qui utilise des KEY des autres totaux

Par exemple, pour un taux pondr, la somme des taux nest pas possible, mais on peut utiliser les sommes des autres totaux:

round(SRU_RP_2020 * (0.25 - SRU_TX_LLS_2020), 4)

Diagnostic

Sur les donnes collectes, des vrifications peuvent tre faites.
Le fichier de configuration xls contient un Tab Diagnostic.
Chaque ligne permet de vrifier une condition et dafficher le diagnostic correspondant.
Ces diagnostics sont ensuite disponibles pour gnrer le rapport.

Exemple : LOG_SRU

Type : Le type du Diagnostic

Les possibilits sont :

  • DIAG : Vrai saffiche en vert, Faux en rouge
  • NOTE : Saffiche en jaune, mais le message varie si Vrai ou Faux
  • TEST : Pour les tests et le debugging

Rapport

Le rapport consiste en :

  • Un fichier xls avec les donnes consolides pour la commune ou le territoire
  • Un fichier csv avec les donnes consolides pour la commune ou le territoire
  • Une page HTML avec des graphiques, des diagnostics, etc.

Rapport CSV

Le rapport CSV ne contient que les mtriques brutes, avec le total et les meta-donnes

Rapport JSON

Les rapports JSON ne contient que les mtriques brutes, avec le total et les meta-donnes. Il y a trois fichiers:

  • Vue selon les communes (_c) : une entree par commune(+ meta), avec les valeurs pour chaque mtrique
  • Vue selon les metriques (_m) : une entree par mtrique, avec les valeurs pour chaque commune (+ meta)

Rapport Excel

Le rapport gnr est considr comme corrompu par Excel, mais il peut le rparer)).
Il contient 3 TAB :

  • TAB Data : les mtriques brutes, avec le total et les meta-donnes
  • TAB Pivot : une transposition de ces donnes
  • TAB Diagnotics : la liste des diagnostics et les valeurs calcules

Rapport HTML

Le rapport HTML est base sur un template, qui permet dajuster la prsentation et les donnes a afficher.

Le fichier template de rapport par dfaut est "input/report_template.html"

Il utilise le moteur MAKO.
